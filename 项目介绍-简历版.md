# Mini Chat Bar - 全栈实时聊天应用

## 项目概述

Mini Chat Bar 是一个基于现代Web技术栈开发的全栈实时聊天应用，支持私聊、群聊、文件传输、AI助手等功能，并提供跨平台桌面端支持。项目采用前后端分离架构，具备完整的用户管理、实时通信、数据持久化等企业级功能。

## 核心技术栈

### 前端技术
- **框架**: Vue 3 (Composition API)
- **构建工具**: Vite 6.3.5
- **状态管理**: Pinia
- **路由管理**: Vue Router 4
- **UI组件**: 自定义组件 + FontAwesome图标库
- **实时通信**: Socket.IO Client
- **桌面端**: Electron 36.8.1
- **样式预处理**: Sass
- **HTTP客户端**: Axios
- **特色功能**: Vue3-Emoji-Picker 表情选择器

### 后端技术
- **运行环境**: Node.js
- **Web框架**: Express.js 5.1.0
- **数据库**: MongoDB + Mongoose ODM
- **实时通信**: Socket.IO 4.8.1
- **身份验证**: JWT (JSON Web Token)
- **密码加密**: bcrypt 6.0.0
- **文件上传**: Multer 2.0.2
- **AI集成**: DeepSeek API
- **跨域处理**: CORS
- **环境配置**: dotenv

### 开发工具
- **包管理**: npm
- **开发服务器**: Vite Dev Server + Nodemon
- **打包构建**: Electron Builder
- **版本控制**: Git
- **代码编辑**: VS Code

## 核心功能实现

### 1. 用户认证系统
- **注册/登录**: 基于JWT的无状态身份验证
- **密码安全**: bcrypt加密存储
- **权限控制**: 路由守卫 + 中间件验证
- **会话管理**: Token自动续期机制

### 2. 实时通信系统
- **WebSocket连接**: Socket.IO双向通信
- **私聊功能**: 点对点消息传递
- **群聊功能**: 房间广播机制
- **在线状态**: 实时用户状态同步
- **消息持久化**: MongoDB存储聊天记录

### 3. 文件传输系统
- **多文件上传**: 支持图片、文档、视频等多种格式
- **文件预览**: 内置图片/视频预览器
- **下载功能**: 安全的文件下载机制
- **存储管理**: 文件元数据管理

### 4. AI助手集成
- **API集成**: DeepSeek AI模型接入
- **智能对话**: 上下文理解与回复
- **异步处理**: 非阻塞AI响应机制

### 5. 桌面端应用
- **Electron封装**: Web应用桌面化
- **跨平台支持**: Windows/macOS/Linux
- **自动更新**: 应用版本管理
- **系统集成**: 原生桌面体验

## 技术架构设计

### 前端架构
```
┌─────────────────┐
│   Vue 3 App     │
├─────────────────┤
│ Pinia Store     │  ← 状态管理
│ Vue Router      │  ← 路由管理
│ Socket.IO       │  ← 实时通信
│ Axios           │  ← HTTP请求
└─────────────────┘
```

### 后端架构
```
┌─────────────────┐
│   Express.js    │
├─────────────────┤
│ JWT Middleware  │  ← 身份验证
│ Socket.IO       │  ← WebSocket服务
│ Mongoose ODM    │  ← 数据库操作
│ Multer          │  ← 文件处理
└─────────────────┘
        ↓
┌─────────────────┐
│    MongoDB      │  ← 数据持久化
└─────────────────┘
```

### 数据模型设计
- **用户模型**: 用户信息、好友关系、头像管理
- **消息模型**: 消息内容、时间戳、发送者/接收者
- **房间模型**: 群聊房间、成员管理、权限控制
- **联系人模型**: 好友列表、最近聊天记录

## 技术难点与解决方案

### 1. 实时消息同步
**难点**：多用户实时消息同步，消息顺序保证
**解决**：Socket.IO事件驱动 + MongoDB时间戳排序

### 2. 文件上传处理
**难点**：大文件上传、多文件处理、文件预览
**解决**：Multer中间件 + 前端文件预览 + 分片上传

### 3. 状态管理复杂性
**难点**：聊天状态、用户状态、UI状态管理
**解决**：Pinia模块化状态管理 + 响应式数据绑定

### 4. 跨平台兼容性
**难点**：Web端与桌面端的兼容性问题
**解决**：Electron主进程配置 + 渲染进程优化
